# User Study 方法论笔记

> 日期: 2026-03-16
> 背景: 晚7点 w2-501 User Study 会议

---

## 什么是 User Study？

**简单说**: 系统地收集真实用户与系统交互的数据，评估可用性、用户体验、验证研究假设。

---

## 为什么需要 User Study？

对于人机协作(HRC)研究：

1. **验证假设** — "共享控制真的比全自主好吗？" 需要真人实验证明
2. **发现问题** — 用户行为可能和预想完全不同
3. **量化指标** — 任务完成时间、错误率、主观信任度评分
4. **论文必备** — HRC领域顶级会议几乎都要求 human evaluation

---

## User Study 常见类型

| 类型 | 目的 | 方法 |
|------|------|------|
| **可用性测试** | 系统好用吗？ | 任务观察 + SUS问卷 |
| **对比实验** | A比B好吗？ | A/B test + 统计检验 |
| **主观评估** | 用户感受如何？ | Likert量表、访谈 |
| **纵向研究** | 长期使用效果 | 多次测试 + 学习曲线 |

---

## User Study 核心要素

### 1. 研究问题
> "在共享控制模式下，用户对机器人的信任度是否高于全自主模式？"

### 2. 实验设计
- 被试数量（通常 15-30 人）
- 分组（between-subjects vs within-subjects）
- 任务设计（典型场景 + 对照场景）

### 3. 数据收集
- **客观指标**: 任务成功率、完成时间、碰撞次数
- **主观指标**: NASA-TLX（负荷）、SUS（可用性）、信任量表

### 4. 统计分析
- t-test / ANOVA 比较组间差异
- Effect size 报告效应大小
- p < 0.05 显著性检验

---

## 对研究的具体价值

### CT介入机器人
- 医生对"共享控制穿刺"的接受度？
- 不同风险等级下用户的干预偏好？

### 制造业 HRC
- 工人对协作机器人的信任建立过程？
- 意图预测准确率对用户安全感的影响？

## 论文中的 User Study 做法

### VLA模型论文（RT-2, π0.5, MoLe-VLA, Helix等）
**实验方法**: 自动化评估，没有人类被试

| 论文 | 评估方式 | 指标 |
|------|----------|------|
| RT-2 | 6000+机器人任务 | 成功率、泛化能力 |
| π0.5 | 真实机器人测试 | 任务完成率 |
| MoLe-VLA | 仿真+真机对比 | 计算效率、成功率 |
| Helix | 双机器人协作任务 | 成功率、实时性 |

**没有 User Study 的原因**:
- 研究重点是模型架构，不是人机交互
- 任务是"机器人执行指令"，不需要人参与

### HRC/数字孪生论文
**这类论文才可能做 User Study！**

### 医疗机器人论文
- 主要是**体模实验**或**尸体实验**
- 少数有**外科医生评估**（主观评分）
- 很少有正式 User Study（伦理审批复杂）

---

## 常见坑

- 被试太少（统计效力不足）
- 任务太简单/太难（天花板/地板效应）
- 指标设计不合理（测不出差异）
- 伦理审批没做（涉及人类被试）

---

## 如果要做 User Study

**建议参考的框架**:

```
研究问题：共享控制 vs 全自主，哪个更适合CT穿刺？

实验设计：
├── 被试：15名医学生/住院医生
├── 分组：within-subjects（每个人测两种模式）
├── 任务：仿真CT穿刺（避开危险区域）
└── 对比：共享控制组 vs 全自主组

数据收集：
├── 客观：穿刺精度、完成时间、碰撞次数
└── 主观：信任量表、NASA-TLX、SUS

统计检验：
└── t-test比较两组差异
```
